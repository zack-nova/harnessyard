# Marked guidance resolution before Run View cleanup

Run View cleanup removes root guidance markers, which also removes the last machine-readable owner identity for any edited marked guidance block. We decided that interactive cleanup must ask users whether to save the current block to authored truth, re-render authored truth before cleanup, or strip markers in place; non-interactive cleanup must expose the same choices through explicit flags and fail closed when unresolved. We chose this over a single force path because the three choices preserve different facts, and hiding that distinction would make normal runtime presentation cleanup look like package authoring drift or silently discard recoverable authoring intent.

Standalone Run View root guidance output outside package installation is explicit: non-interactive output should use output language such as `--output` rather than force language. Package installation remains the normal happy path that may compose guidance for the newly installed package and clean markers automatically so Run View users can start using the installed guidance immediately. Installation output is incremental in Run View because existing markerless root guidance is presentation text without owner identity; recomposing all packages would either duplicate existing presentation text or risk replacing user-edited presentation content. Incremental install guidance is appended with stable separation rather than reordering or deduplicating existing markerless text.
